## 7. Decisions & why
- **REST not tRPC:** simpler backend, manual DTO sync cost acceptable, easier to onboard. Type safety traded for simplicity.
- **Radix UI standalone (no shadcn registry):** lighter bundle, full control, zero lock-in to third-party component library updates.
- **axios + TanStack Query:** familiar, powerful caching/dedup, optimistic updates, great devx.
- **Supabase Auth:** RLS-safe (anon key public), SSR-safe (@supabase/ssr), JWT role-aware, easy multi-tenant.
- **Biome single tool:** fast, good defaults, replaces ESLint + Prettier, config-free for this project.
- **Server Components default:** Next.js 16 best practice, reduces JS bundle, simpler data flow.
- **CSS custom properties:** consistent theming, override per tenant (F1), smaller CSS than Tailwind config overrides.
- **decimal.js:** match backend precision, prevent float rounding errors on money/decimals.
- **Light-only theme:** professional medical application, no dark mode complexity.
- **Manual type mirror:** avoids monorepo/shared package complexity; cost is manual sync on back DTOs change (annotate in PR).

## 10. Gotchas & landmines
- **Type sync manual:** update lib/api/types.ts by hand when backend DTO changes; no auto-generation. Annotate PR in backend repo.
- **NEXT_PUBLIC_API_URL must include /api:** e.g. `https://api.lab.local/api`, not `https://api.lab.local` (endpoints are GET /api/ordenes, not GET /ordenes).
- **Middleware (proxy.ts) ~50-150ms latency:** Supabase JWT validation on every request. Optimize via KV cache if needed (F9+).
- **No dark mode:** tokens defined light-only; don't add dark: variants.
- **decimal.js precision 28 globally:** match backend; keep money as strings in JSON wire format, never JSON.stringify(new Decimal()).
- **Supabase RLS:** anon key is public-safe (RLS protects); NEVER expose service_role key with NEXT_PUBLIC_ prefix.
- **Next.js 16 breaking changes:** read https://nextjs.org/docs before writing unfamiliar APIs. Server Components vs Client different rules.
- **Query staleTime:** default 30s (refetch if older); configure per useQuery if needed.
- **Lab branding server-side:** /lab-config endpoint blocks page load; cache aggressively or move to layout-level fetch.
- **Tenant slug routing (F1):** slug = routing/branding ONLY. Authorization = JWT labId. Slug mismatch in middleware = silent redirect to user's slug (prevent data flash).
- **CORS required:** backend must list frontend domain (Vercel URL + localhost:3000 for dev) in CORS_ALLOWED_ORIGINS.
- **Signature_pad:** canvas-based, needs 'use client'; initialize in useEffect after hydration to avoid SSR mismatch.
- **React 19 hooks:** use latest TanStack Query v5, avoid deprecated useFetcher etc.
